JWT Token Structure
// Header
{ "alg": "HS256", "typ": "JWT" }

// Payload Claims
{
  ClaimTypes.Name: "username",
  ClaimTypes.NameIdentifier: "userId",
  ClaimTypes.Role: "Admin",
  "exp": 480 phút
}

// Signing
HMAC-SHA256(SecretKey)
Password Hashing — PBKDF2
// Khi tạo tài khoản
salt = RandomBytes(16)
hash = PBKDF2(
  password, salt,
  iterations: 100_000,
  prf: HMAC-SHA256,
  outputLen: 32 bytes
)

// Lưu database
User.Password = Base64(hash)
User.Salt     = Base64(salt)

4C. Soft Delete Pattern
Áp dụng cho:
  • Product (IsDeleted)
  • Category (IsDeleted)
  • Supplier (IsDeleted)
Cơ chế:
  • DELETE → set IsDeleted = true
  • Global Query Filter tự động lọc
  • OrderDetail vẫn giữ lịch sử đơn hàng
  • Restrict FK nếu có trong OrderDetail
